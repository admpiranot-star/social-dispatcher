📊 ARQUITETURA
═══════════════════════════════════════════════════════════════════════════════

Nexus Publisher
     ↓ (POST /api/dispatch com artigo)
Social Dispatcher (porta 3302)
     ├→ Dispatcher (orquestrador)
     │   ├→ INSERT posts
     │   └→ Enqueue jobs
     │
     ├→ BullMQ Queues (Redis)
     │   ├→ facebook-distribution (workers: 120 req/min)
     │   ├→ instagram-distribution (workers: 120 req/min)
     │   └→ whatsapp-distribution (workers: 80 req/min)
     │
     ├→ Workers (BullMQ)
     │   ├→ FacebookWorker (Graph API)
     │   ├→ InstagramWorker (Graph API)
     │   └→ WhatsAppWorker (Cloud API)
     │
     ├→ PostgreSQL (auditoria)
     │   ├→ social_accounts (credenciais)
     │   ├→ posts (artigos enfileirados)
     │   ├→ job_logs (rastreamento)
     │   ├→ metrics (engagement)
     │   └→ ab_tests (A/B testing)
     │
     └→ Redis (cache)
         ├→ token:accountId (cache com TTL)
         └→ rate-limit:accountId (janela 1min)

Meta Graph API (v19.0)
  ├→ /v19.0/{pageId}/feed (Facebook)
  ├→ /v19.0/{businessAccountId}/media (Instagram)
  └→ /v19.0/{phoneNumberId}/messages (WhatsApp)

🔌 ENDPOINTS API
═══════════════════════════════════════════════════════════════════════════════

POST /api/dispatch
  Descrição: Enfileirar post para distribuição
  Auth: Bearer Token (change-this-in-production)
  Body: {
    title, link, channels, category, priority, imageUrl, metadata
  }
  Response: 202 Accepted + array de jobs

GET /api/status/:postId
  Descrição: Verificar status de um post
  Response: Array de job_logs + métricas

GET /api/metrics
  Descrição: Métricas das filas BullMQ
  Response: { queues: { facebook, instagram, whatsapp } }

GET /api/health
  Descrição: Health check
  Response: { status: "ok", timestamp: "..." }
